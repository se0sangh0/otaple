# Otaku Trip Assistant

여행지와 일정을 넣으면 오타쿠 여행에 필요한 후보를 자동으로 추려주는 웹 앱입니다.

- 입력: 도시, 여행 기간, 여행 강도, 관심 태그
- 옵션: 실시간 웹 수집 on/off, 수집 최대 개수
- 출력:
  - 해당 도시의 서브컬쳐 이벤트/상설 매장/콜라보 카페 후보
  - 실시간 이벤트 장르/IP 보드 (예: 주술회전/홀로라이브/니지산지)
  - 일자별 자동 일정안 (지역 묶음 중심)
  - 마크다운 내보내기

## 실행

별도 빌드 없이 정적 파일로 동작합니다.

1. `otaku-trip-assistant/index.html` 브라우저로 열기
2. 도시/일정 입력 후 `여행 플랜 생성`

## 여행지 입력

- 도쿄/오사카/서울은 기본 데이터셋이 함께 적용됩니다.
- 그 외 도시명도 입력 가능하며, 실시간 웹 수집 결과 위주로 추천합니다.

## 데이터 구조

데이터는 `data.js`의 `BASE_SPOTS` 배열에 있습니다.

```js
{
  id: "tokyo-animejapan",
  city: "tokyo",
  type: "event", // event | store | cafe | complex
  tags: ["anime", "event"],
  schedule: {
    kind: "recurring", // always | rolling | range | recurring
    months: [3],
    note: "매년 3월 말 개최 경향"
  },
  officialCheckRequired: true,
  source: "https://...",
  mapsQuery: "Tokyo Big Sight"
}
```

## 추천 로직(요약)

1. 도시 매칭
2. 여행 기간과 장소 일정 겹침 판단
3. 관심 태그/장소 유형/콜라보 우선 옵션 기반 점수 계산
4. 점수 상위 후보를 일자별로 배분해 자동 일정안 생성

## 실시간 수집

- 로그인/인증 없이 공개 피드만 사용합니다.
- 소스: Google News RSS 검색 결과를 `rss2json`으로 파싱
- 수집 대상:
  - 애니/게임/VTuber 이벤트 키워드
  - 콜라보 카페 키워드
  - 전시/특설전/팝업 키워드
  - IP 집중 키워드(주술회전/홀로라이브/니지산지)
- 결과 카드에 `실시간 수집` 일정 라벨이 붙습니다.
- 결과 상단에 장르/IP별 묶음 보드가 함께 표시됩니다.
- 연례/기사 기반 정보는 변동 가능하므로 공식 링크 재확인이 필요합니다.

## 주의

- 연례/순환형 이벤트는 변동이 많으므로 `공식 일정 확인 필요` 배지가 붙습니다.
- 최종 예약/운영 시간은 카드의 공식 링크에서 재확인하세요.
